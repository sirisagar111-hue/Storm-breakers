# Project Overview

The **Blockchain-Based Secure Health Record Vault** is a decentralized healthcare platform designed to securely store and manage patient medical records such as:

* X-rays
* MRI scans
* Prescriptions
* Lab reports
* Discharge summaries

The system gives patients complete ownership and control over their medical data using blockchain technology, encryption, and smart contracts.

Unlike traditional healthcare systems where hospitals control patient records, this platform ensures that only the patient can decide who gets access to their medical information.

---

# Problem Statement

Current healthcare systems face several major problems:

## 1. Fragmented Medical Records

Medical records are stored separately across hospitals, clinics, and laboratories. This causes:

* Missing records
* Delayed treatments
* Repeated medical tests
* Difficulty transferring records

## 2. Lack of Patient Control

Patients do not fully control who can access their health data. Hospitals and third parties often manage permissions instead.

## 3. Security Risks and Data Breaches

Centralized hospital databases are vulnerable to:

* Hacking
* Unauthorized access
* Data tampering
* Exposure of sensitive patient information

---

# Proposed Solution

This project creates a secure digital vault where:

* Medical files are encrypted before storage
* Blockchain stores secure record metadata
* Patients can grant or revoke access to doctors
* Smart contracts enforce access permissions
* Every activity is transparently logged

The system ensures:

* Privacy
* Transparency
* Tamper-proof records
* Patient ownership of healthcare data

# System Workflow

1. Patient registers and logs into the system.
2. Medical records are uploaded and encrypted.
3. Files are stored securely using IPFS/cloud storage.
4. Blockchain stores file hashes and permissions.
5. Doctors request access to records.
6. Patients approve or reject requests.
7. Smart contracts enforce permissions automatically.
8. All actions are permanently logged for transparency.
